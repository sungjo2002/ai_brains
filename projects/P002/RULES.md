# P002 작업 규칙

## 기본 원칙

- P002는 1인 기업 AI 사무실 프로젝트다.
- P001과 혼동하지 않는다.
- P001 관련 기준은 P001 문서를 우선한다.
- P002는 connect-ai를 참고하지만 그대로 복사하는 프로젝트가 아니다.
- 성조님 업무 방식에 맞춘 확장형 시스템으로 만든다.

## 작업 판단 순서

1. 현재 대화의 사용자 지시
2. `projects/P002/CURRENT.md`
3. `projects/P002/RULES.md`
4. `projects/P002/ROADMAP.md`
5. `projects/P002/STRUCTURE.md`
6. `projects/P002/MEMORY.md`
7. `projects/P002/TOOLS.md`
8. 참고 저장소 connect-ai 실제 코드
9. 과거 대화/메모

## 수정 전 규칙

실제 GitHub 파일 수정 전에는 가능한 경우 다음을 먼저 정리한다.

- 수정 대상
- 수정 이유
- 수정 범위
- 생성/수정 파일명
- 확인 방법

## 승인 필요한 작업

아래 작업은 자동 실행하지 말고 사용자 승인 후 실행한다.

- Cafe24 상품 등록
- 상품 가격 수정
- 상품 삭제
- 고객 답변 발송
- 광고비 집행
- 급여/근태 확정
- 서버 파일 수정
- GitHub 코드 대량 변경
- 자동 git push 설정 변경
- API 키/토큰 저장

## 자동화 가능 작업

아래 작업은 초안/정리 중심으로 자동화 가능하다.

- 문서 요약
- 상품명/SEO 초안
- 고객 답변 초안
- 이미지 생성 프롬프트 작성
- GitHub 작업 정리
- 서버 로그 요약
- 지식 창고 문서 분류
- 작업 기록 저장

## 기억 규칙

- 중요한 기억은 사용자 승인 후 저장한다.
- 오래된 기준과 최신 기준이 충돌하면 최신 기준을 우선한다.
- 출처가 있는 기억을 우선한다.
- 프로젝트별 기억을 분리한다.

## 언어/응답 규칙

- 기본 답변은 한국어로 한다.
- 요약을 먼저 제공한다.
- 복잡한 내용은 단계별로 설명한다.
- 명령어와 코드는 복사 가능한 코드블록으로 제공한다.

## 보안 원칙

- API 키, 토큰, 비밀번호는 문서에 평문 저장하지 않는다.
- 실행 권한이 있는 자동화는 사용자 승인 단계를 둔다.
- 삭제/발송/등록/결제성 작업은 기본적으로 막고 승인 후 진행한다.
