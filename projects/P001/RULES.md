# RULES

이 파일은 P001 프로젝트의 고정 규칙을 기록한다.

## 파일명 규칙

- PC 수정본: `pc_번호_영문설명.zip`
- 모바일 수정본: `mobile_번호_영문설명.zip`
- 서버 수정본: `server_번호_영문설명.zip`
- PC, 모바일, 서버 수정본은 각각 따로 만든다.
- 불필요한 캐시 파일은 포함하지 않는다.

## 수정 실행 규칙

분석 요청과 실제 수정 요청을 구분한다.

분석 요청 예시:

```text
하자
할까
정리하자
순서대로 하자
어떨까
```

실제 수정 요청 예시:

```text
수정해
적용해
파일 만들어줘
생성해
ZIP 만들어줘
저장해
```

실제 수정 전에는 먼저 `수정 대기`를 출력하고 수정 대상, 이유, 범위, 파일명, 확인 방법을 짧게 정리한다.

## 작업 기준

- 최신 상태는 `CURRENT.md`를 기준으로 본다.
- 다음 작업은 `TASKS.md`를 기준으로 본다.
- 작업 기록은 `HISTORY.md`에 남긴다.
- 중요한 결정은 `DECISIONS.md`에 남긴다.
- 실제 파일과 사용자 확인 결과가 저장소 기록보다 최신이면 실제 파일과 확인 결과를 우선한다.

## 금지

- 기능 조건, 권한 조건, 저장 조건, 동기화 조건, 계산 조건은 임의로 바꾸지 않는다.
- 서버 방어 로직과 화면 숨김을 혼동하지 않는다.
