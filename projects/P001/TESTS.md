# TESTS

이 파일은 P001 프로젝트 수정 후 확인할 테스트 기준을 기록한다.

## 사업자/사업장 동기화 테스트

1. PC1에서 A사업자에 B사업장을 추가한다.
2. PC1에서 동기화한다.
3. PC2에서 동기화한다.
4. PC2에서 다른 저장 동작 없이 A사업자 아래 B사업장이 바로 보이는지 확인한다.
5. 사업장 수정 창에서 소속 사업자를 다른 사업자로 변경 가능한지 확인한다.
6. 변경 후 PC2와 모바일에 반영되는지 확인한다.

## 근로자/차량/근태 동기화 테스트

- PC1에서 근로자 등록 후 동기화하고 PC2에서 표시 확인
- PC2에서 근로자 등록 후 동기화하고 PC1에서 표시 확인
- 모바일에서 근로자 등록 후 PC1/PC2 동기화 표시 확인
- 모바일에서 근태 입력 후 PC1/PC2 동기화 반영 확인
- PC1에서 차량 등록 후 PC2 동기화 표시 확인
- PC2에서 차량 등록 후 PC1 동기화 표시 확인

## 삭제 동기화 테스트

1. PC1에서 테스트 근로자를 삭제한다.
2. 오류 없이 바로 사라지는지 확인한다.
3. PC1에서 동기화한다.
4. 동기화 실패 표시가 없는지 확인한다.
5. PC2에서 동기화한다.
6. PC2에서도 삭제됐는지 확인한다.
7. 모바일에서도 삭제됐는지 확인한다.

## 공휴일 자동갱신 테스트

1. PC1에서 공휴일 자동갱신 설정을 저장한다.
2. PC1에서 동기화한다.
3. PC2에서 동기화한다.
4. PC2 설정에 반영되는지 확인한다.
5. 근태관리/급여관리 달력에서 공휴일 표시를 확인한다.

## PC_A / PC_B 한 컴퓨터 테스트 주의

- 테스트 폴더를 나눈다.
- 로컬 데이터는 분리되어야 한다.
- 백업 위치는 가능하면 다르게 둔다.
- PC_A에서 동기화 전 등록한 자료가 PC_B에 바로 보이면 같은 로컬 DB를 쓰는 상태라 테스트가 정확하지 않다.
