# DECISIONS

이 파일은 P001 프로젝트의 중요한 결정 사항을 기록한다.

## 2026-05-20 급여 시간 환산 기준 결정

- 사업장별 시간 환산 기준은 프로그램 설정값으로 관리한다.
- 모든 업무값과 설정값은 기본적으로 동기화 대상으로 본다.
- 사업장별 시간 환산 기준도 동기화 대상이다.
- 기본 사업장 환산표는 프로그램 코드에 영구 고정하지 않고, 불러오기 기능으로 설정 데이터에 저장한 뒤 동기화 대상으로 관리한다.
- 사업장별 시간 환산 기준을 불러오면 근무사업장 마스터와 사업장 검색 목록에도 연결되어야 한다.
- 급여관리 자동 적용은 기존 수기 입력값을 보호하기 위해 버튼 방식으로 시작한다.
- 현재 급여관리 표가 `기본 / 연장 / 심야` 중심이므로, 급여명세표 확인 전까지 특근/특근연장/공휴특근/주휴를 별도 행으로 확정하지 않는다.
- 다음 수정 전에는 실제 급여명세표 또는 정산표를 확인해 항목 표시 방식을 확정한다.

## 2026-05-20 사업장별 계산 방식 결정

- 광명산업/무진은 시간대별 환산표와 마감환산표를 함께 가진다.
- 광명산업/무진은 6일 근무 기준이며, 월~금 정상 근무 조건을 만족하면 일요일은 주휴수당 지급일로 본다.
- 광명산업/무진에서 일요일 실제 근무가 있는 경우 주휴수당과 휴일근무를 구분해야 한다.
- 지스틸/강동은 날짜별 월간 환산표 방식이다. 일요일 자동 제외가 아니라 월간표에 적힌 날짜별 값 그대로 적용한다.
- 현우는 평일/토요일/공휴일, 주간/야간 기준의 시간대별 환산표 방식이다.
- 에코인슈텍/양산은 항목별 배율 계산 방식이다.

## 2026-05-20 저장소 구조 결정

- 저장소는 특정 프로젝트명 중심이 아니라 프로젝트 ID 중심으로 관리한다.
- 상위 폴더명에는 특정 작업명을 쓰지 않는다.
- 첫 번째 프로젝트는 `projects/P001/`로 관리한다.
- 최신 상태는 `CURRENT.md` 하나만 기준으로 본다.
- 다음 작업은 `TASKS.md` 하나만 기준으로 본다.
- 여러 AI와 코딩 도구가 같은 순서로 읽을 수 있게 `AGENTS.md`를 루트 기준 파일로 사용한다.

## 2026-05-19 동기화 방향 결정

- PC1을 메인 PC로 보지 않는다.
- 어느 PC 또는 모바일에서 등록/수정/삭제하더라도 서버를 기준으로 다른 PC와 모바일에 반영되는 구조로 잡는다.
- 사용자는 동기화 버튼을 기존처럼 하나만 사용한다.
- 내부적으로는 저장/불러오기/병합/삭제 반영을 자동 처리한다.
- 사용자 화면에는 복잡한 버튼을 늘리지 않는다.

## 2026-05-19 이미지 동기화 결정

서버 사진 동기화 보강에서 제외할 대상은 다음으로 확정한다.

- 신분증 원본 이미지
- 여권 원본 이미지
- 신분증 보정본 이미지
- 여권 보정본 이미지

근로자 얼굴 사진은 제외 대상과 별도이며 실제 구현 상태를 확인해야 한다.
