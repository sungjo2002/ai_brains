# RULES

이 파일은 저장소 전체 공통 규칙을 기록한다.

## 공통 운영 원칙

- 이 저장소는 여러 AI와 코딩 도구가 공통으로 사용하는 작업 저장소다.
- 특정 AI, 특정 프로젝트, 특정 프로그램에 종속되지 않는다.
- 프로젝트는 `projects/P001`, `projects/P002`처럼 번호로 관리한다.
- 프로젝트 이름은 `PROJECTS.md`와 각 프로젝트의 `META.md` 안에만 기록한다.

## 기준 파일 원칙

- 전체 시작 기준은 `AGENTS.md`다.
- 프로젝트 목록은 `PROJECTS.md`다.
- 저장소 전체 변경 이력은 `CHANGELOG.md`다.
- 각 프로젝트의 최신 상태는 `CURRENT.md` 하나만 기준으로 본다.
- 각 프로젝트의 다음 작업은 `TASKS.md` 하나만 기준으로 본다.
- 각 프로젝트의 고정 규칙은 `RULES.md`에 둔다.

## 수정 전 규칙

실제 수정, 생성, 저장 요청이 있으면 먼저 아래 항목을 짧게 정리한다.

```text
수정 대기
- 수정 대상:
- 수정 이유:
- 수정 범위:
- 생성/수정 파일명:
- 확인 방법:
```

## 보안 원칙

운영 설정, 인증 정보, 데이터베이스, 개인정보 원본 파일, 백업 데이터는 저장하지 않는다.

## 코딩 작업 원칙

- 실제 코드가 저장될 경우 각 프로젝트의 `source/` 아래에 둔다.
- 산출물 또는 ZIP 수정본은 각 프로젝트의 `packages/` 아래에 둔다.
- 오래된 기록은 각 프로젝트의 `archive/` 아래에 둔다.
- 코드 수정 후에는 해당 프로젝트의 `COMMANDS.md`와 `TESTS.md` 기준으로 확인한다.
