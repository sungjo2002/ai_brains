# workforce README

이 파일은 인력관리 프로그램의 현재 작업 상태를 기록한다.

## 최신 상태

- PC 최신 수정본: pc_1_all_pages_column_resize_default_width_fix.zip
- 서버 최신 수정본: server_10_attendance_clear_tombstone_fix.zip
- 모바일 최신 수정본: mobile_16_attendance_retry_queue_fix.zip
- PC 수정본 번호는 pc_69 이후 새 구간으로 다시 시작했다.
- 현재 PC 작업은 PC 화면 안정화와 표 컬럼 사용성 정리 중심으로 진행 중이다.
- 깜빡임 문제는 일부 개선됐지만 완성된 홈 화면의 잔깜빡임은 보류 상태다.
- pc_1에서는 근태관리와 급여관리를 제외한 표 페이지의 컬럼 조절 기능과 기본 컬럼 폭 재계산을 진행했다.

## 번호 규칙

- PC 수정본은 새 구간 기준 pc_1부터 다시 시작한다.
- 다음 PC 수정본은 pc_2부터 이어간다.
- 모바일 수정본은 기존 번호를 유지한다.
- 서버 수정본은 기존 번호를 유지한다.

## 완료

- 배너 관련 문제 일부 개선 완료.
  - 홈 배너 빈 배지 박스 제거.
  - 배너 늦게 표시되는 느낌 개선.
  - 페이지별 배너 정상 표시 확인.
- 왼쪽 메뉴 클릭 시 흰 화면 깜빡임은 상당 부분 개선 완료.
- 로그인 후 홈 화면이 뜨기 전 빈 화면 깜빡임은 개선 완료.
- 로그인 화면 디자인 정리 완료.
  - 2번 시안 방향 적용.
  - 기존 프로그램 로고 사용.
  - 제목을 관리자 로그인으로 변경.
  - 설명 문구 삭제.
  - 전체 크기 축소.
  - 로고 크기 재조정.
- 급여관리 화면 보강 완료.
  - 우측 이중 스크롤 제거 보강.
  - 급여표 로딩 속도 개선.
  - 급여관리 전체 화면이 외부 스크롤처럼 움직이는 문제 보강.
- 홈화면 인원 목록에서 표시번호를 맨 앞으로 이동하고 제목을 번호로 변경 완료.
- 근로자 목록에서 이름, 영문이름 칸 폭이 너무 좁은 문제 1차 보강 완료.
- pc_66에서 표 전체 폭 고정과 하단 가로 스크롤 방지 보강을 진행했다.
- pc_67에서 컬럼 조절 시 잡은 칸과 바로 옆 칸만 움직이도록 보강했다.
- pc_68에서 컬럼 폭을 app_settings.json의 local_ui_settings에 저장하도록 보강했다.
- pc_69에서 초기 레이아웃 보정이 저장값을 덮어쓰지 않도록 재적용 로직을 보강했다.
- pc_1에서 공통 컬럼 조절 관리 파일 src/table_column_manager.py를 추가했다.
- pc_1에서 홈화면과 근로자 목록의 컬럼 조절 방식을 공통 관리 방식으로 연결했다.
- pc_1에서 사업자 관리, 차량 관리, 환경설정의 표 컬럼 조절을 적용했다.
- pc_1에서 각 표의 기본 컬럼 폭을 헤더와 셀 글자 길이 기준으로 다시 계산하도록 했다.
- pc_1에서 조절한 컬럼 폭을 local_ui_settings와 QSettings에 같이 저장하도록 했다.

## 진행 중

- pc_1 기준 사용자 실제 확인 대기.
- 확인 필요 항목:
  - 홈화면 표 기본 칸 크기와 글자 가독성.
  - 근로자 관리 표 기본 칸 크기와 글자 가독성.
  - 사업자 관리, 차량 관리, 환경설정 표의 컬럼 조절 가능 여부.
  - 컬럼 조절 후 프로그램 재시작 시 저장값 복원 여부.
  - 컬럼 조절 시 잡은 칸과 바로 옆 칸 중심으로 움직이는지 여부.
  - 기본 상태에서 하단 가로 스크롤이 생기지 않는지 여부.

## 다음 작업

1. pc_1을 실제 실행해 홈화면/근로자/사업자/차량/환경설정 표를 확인한다.
2. 컬럼 기본 폭이 넓거나 좁은 페이지가 있으면 해당 표만 기본 폭을 재조정한다.
3. 재시작 후 저장값 복원이 실패하는 표가 있으면 해당 키와 저장 시점을 다시 확인한다.
4. 근태관리와 급여관리는 이번 컬럼 조절 작업에서 제외했으므로 별도 요청이 있을 때만 건드린다.
5. 확인 완료 후 다음 PC 수정본은 pc_2로 진행한다.

## 보류

- 완성된 홈 화면에서 보이는 잔깜빡임은 다음으로 보류.
- 시작/종료 속도 개선은 레이아웃/표 사용성 정리 후 진행.
- 모바일 고도화, 서버 동기화 고도화, OCR 고도화는 PC 1차 정리 후 진행.

## 오류/주의

- pc_64는 표 컬럼 자동 보정과 사용자 조절이 충돌하여 실패로 판단한다.
- pc_65는 홈화면 하단 스크롤과 현재 상태 컬럼 조절 불가 문제가 남아 있었다.
- pc_66은 여러 컬럼이 같이 움직이는 부작용이 있었다.
- pc_67은 재시작 후 컬럼 폭 저장 복원이 되지 않았다.
- pc_68은 저장은 시도했지만 초기 레이아웃 보정에 의해 저장값이 밀리는 문제가 있었다.
- pc_69도 사용자 확인 기준 재시작 후 저장값 복원이 되지 않았다.
- pc_1은 공통 컬럼 조절 방식으로 다시 정리한 첫 수정본이므로 실제 화면 확인이 필요하다.
- 표 컬럼 조절 수정 시 자동 폭 보정이 사용자의 마우스 조절 중 끼어들면 안 된다.
- 표 전체 폭은 화면 안에 고정해야 하며, 사용자가 조절하는 컬럼 경계만 움직여야 한다.
- 경계선은 보이게 하되 기존 표 색보다 진하면 안 된다.
- 기능 조건, 저장 조건, 동기화 조건, 계산 조건, 권한 조건은 건드리지 않는다.
- 근태관리와 급여관리는 이번 전체 컬럼 조절 작업에서 제외한다.
- 저장소 기록보다 실제 업로드 파일과 사용자가 확인한 결과를 우선한다.
