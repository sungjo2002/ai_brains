# workforce README

이 파일은 인력관리 프로그램의 현재 작업 상태를 기록한다.

## 최신 상태

- PC 최신 수정본: pc_69_table_column_restore_after_layout_fix.zip
- 서버 최신 수정본: server_10_attendance_clear_tombstone_fix.zip
- 모바일 최신 수정본: mobile_16_attendance_retry_queue_fix.zip
- 현재 PC 작업은 PC 화면 안정화와 사용성 정리 중심으로 진행 중이다.
- 깜빡임 문제는 일부 개선됐지만 완성된 홈 화면의 잔깜빡임은 보류 상태다.
- 표 컬럼 조절 작업은 홈화면/근로자 목록에서 pc_66~pc_69까지 보강을 진행했으나, 사용자가 재시작 후 저장값 복원 문제를 확인했다.
- 다음 작업부터 PC 수정본 번호는 새 구간으로 다시 시작한다.
- 다음 PC 수정본 번호: pc_1
- 다음 작업은 근태관리와 급여관리를 제외한 모든 페이지 표 컬럼 조절 기능과 기본 컬럼 폭 재계산이다.

## 번호 규칙

- PC 수정본은 기존 pc_69 이후부터 새 구간으로 다시 시작한다.
- 다음 PC 수정본 파일명은 pc_1_all_pages_column_resize_default_width_fix.zip 기준으로 진행한다.
- 모바일 수정본은 기존 번호를 유지한다.
- 서버 수정본은 기존 번호를 유지한다.

## 완료

- 배너 관련 문제 일부 개선 완료.
  - 홈 배너 빈 배지 박스 제거.
  - 배너 늦게 표시되는 느낌 개선.
  - 페이지별 배너 정상 표시 확인.
- 왼쪽 메뉴 클릭 시 흰 화면 깜빡임은 상당 부분 개선 완료.
- 로그인 후 홈 화면이 뜨기 전 빈 화면 깜빡임은 개선 완료.
- 로그인 화면 디자인 정리 완료.
  - 2번 시안 방향 적용.
  - 기존 프로그램 로고 사용.
  - 제목을 관리자 로그인으로 변경.
  - 설명 문구 삭제.
  - 전체 크기 축소.
  - 로고 크기 재조정.
- 급여관리 화면 보강 완료.
  - 우측 이중 스크롤 제거 보강.
  - 급여표 로딩 속도 개선.
  - 급여관리 전체 화면이 외부 스크롤처럼 움직이는 문제 보강.
- 홈화면 인원 목록에서 표시번호를 맨 앞으로 이동하고 제목을 번호로 변경 완료.
- 근로자 목록에서 이름, 영문이름 칸 폭이 너무 좁은 문제 1차 보강 완료.
- pc_66에서 표 전체 폭 고정과 하단 가로 스크롤 방지 보강을 진행했다.
- pc_67에서 컬럼 조절 시 잡은 칸과 바로 옆 칸만 움직이도록 보강했다.
- pc_68에서 컬럼 폭을 app_settings.json의 local_ui_settings에 저장하도록 보강했다.
- pc_69에서 초기 레이아웃 보정이 저장값을 덮어쓰지 않도록 재적용 로직을 보강했다.

## 진행 중

- 홈화면과 근로자 목록의 표 컬럼 조절 방식 정리 중.
- 현재 pc_69 기준 사용자 확인 문제:
  - 컬럼 수정 후 프로그램을 재시작하면 다시 기본값으로 돌아감.
  - 저장값 복원 흐름이 아직 최종 성공이 아니다.
- 다음 작업은 개별 보강보다 전체 표 컬럼 기준을 다시 잡는 방향으로 진행한다.

## 다음 작업

1. 다음 수정명은 pc_1_all_pages_column_resize_default_width_fix.zip 기준으로 진행한다.
2. 근태관리와 급여관리는 이번 컬럼 조절 작업에서 제외한다.
3. 홈화면과 근로자 관리를 포함해 나머지 표가 있는 모든 페이지에 컬럼 조절 기능을 적용한다.
4. 각 페이지의 기본 컬럼 폭을 새로 계산해 글자가 잘 보이도록 한다.
5. 이름, 영문이름, 연락처, 상태, 사업장처럼 글자 잘림이 중요한 칸은 기본 폭을 우선 확보한다.
6. 번호, 성별, 짧은 상태값처럼 짧은 칸은 필요한 만큼만 배정한다.
7. 표 전체 폭은 화면 안에 맞추고 기본 상태에서 하단 가로 스크롤이 생기지 않게 한다.
8. 컬럼 조절은 사용자가 잡은 칸과 옆 칸 중심으로 움직이게 한다.
9. 조절한 컬럼 폭은 PC 로컬 설정에 저장하고 재시작 후 복원한다.
10. 자동 화면 맞춤 함수가 사용자가 저장한 컬럼 폭을 덮어쓰지 않게 한다.
11. 표 경계선은 보이게 하되 기존 표 색보다 진하지 않은 연한 색으로 유지한다.

## 보류

- 완성된 홈 화면에서 보이는 잔깜빡임은 다음으로 보류.
- 시작/종료 속도 개선은 레이아웃/표 사용성 정리 후 진행.
- 모바일 고도화, 서버 동기화 고도화, OCR 고도화는 PC 1차 정리 후 진행.

## 오류/주의

- pc_64는 표 컬럼 자동 보정과 사용자 조절이 충돌하여 실패로 판단한다.
- pc_65는 홈화면 하단 스크롤과 현재 상태 컬럼 조절 불가 문제가 남아 있었다.
- pc_66은 여러 컬럼이 같이 움직이는 부작용이 있었다.
- pc_67은 재시작 후 컬럼 폭 저장 복원이 되지 않았다.
- pc_68은 저장은 시도했지만 초기 레이아웃 보정에 의해 저장값이 밀리는 문제가 있었다.
- pc_69도 사용자 확인 기준 재시작 후 저장값 복원이 되지 않았다.
- 표 컬럼 조절 수정 시 자동 폭 보정이 사용자의 마우스 조절 중 끼어들면 안 된다.
- 표 전체 폭은 화면 안에 고정해야 하며, 사용자가 조절하는 컬럼 경계만 움직여야 한다.
- 경계선은 보이게 하되 기존 표 색보다 진하면 안 된다.
- 기능 조건, 저장 조건, 동기화 조건, 계산 조건, 권한 조건은 건드리지 않는다.
- 근태관리와 급여관리는 이번 전체 컬럼 조절 작업에서 제외한다.
- 저장소 기록보다 실제 업로드 파일과 사용자가 확인한 결과를 우선한다.
