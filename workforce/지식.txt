# workforce 지식

이 파일은 인력관리 프로그램의 고정 규칙, 기본 명령어, 주소 기준, 파일명 규칙, 판단 기준을 기록한다.

## 저장소 기준

- 저장소: `sungjo2002/ai_brains`
- 앞으로 대화에서 별도 설명 없이 “저장소”라고 하면 GitHub 저장소 `sungjo2002/ai_brains`를 의미한다.
- 모든 사적인 대화가 아니라 실제 작업 기준/작업 기록/완료 내용/다음 작업만 저장한다.
- 저장소 정리, README.txt 갱신, 지식.txt 갱신, 작업기록 반영은 사용자가 “저장해”, “저장소 정리해”, “README 정리해”, “지식에 추가해”처럼 명확히 요청했을 때만 진행한다.
- 프로그램 수정 중에는 자동으로 저장소를 갱신하지 않는다.

## 파일명 규칙

- PC 수정본: `pc_번호_영문설명.zip`
- 모바일 수정본: `mobile_번호_영문설명.zip`
- 서버 수정본: `server_번호_영문설명.zip`
- PC/모바일/서버 수정본은 각각 따로 만든다.
- ZIP 안에는 불필요한 최상위 바깥 폴더를 만들지 않는다.
- `__pycache__`, `.pyc` 파일은 제거한다.

## 분석 기준

- 항상 현재 상태와 지난 기준을 둘 다 확인한다.
- 현재 상태는 사용자가 방금 올린 화면, 현재 적용된 ZIP, 현재 서버/모바일/PC 동작 결과를 기준으로 본다.
- 지난 기준은 이전 정상 상태, 이전 수정본, README.txt 작업기록, 사용자가 이미 확인한 결과를 기준으로 본다.
- 오류 분석은 현재 현상만 보고 판단하지 말고, 이전 정상 기준과 비교해서 무엇이 달라졌는지 먼저 찾는다.
- 저장소 내용만 보지 않고, 실제 업로드 파일 상태와 사용자가 확인한 실행 결과를 함께 본다.
- 저장소 기록이 현재 파일 상태와 다르면 현재 업로드 파일과 사용자가 확인한 결과를 우선하고, 사용자가 저장/정리를 요청했을 때만 저장소 README.txt를 갱신한다.

## 수정 전 규칙

실제 수정/생성 요청이 있으면 먼저 아래 형식으로 짧게 정리한다.

```text
수정 대기
- 수정 대상:
- 수정 이유:
- 수정 범위:
- 생성/수정 파일명:
- 확인 방법:
```

그 다음 실제 작업을 진행한다.

## 콘솔 명령어 기준

- 콘솔 명령어는 압축을 푼 뒤 파일이 이미 작업 폴더에 들어간 상태를 기준으로 제공한다.
- 긴 압축 해제/백업/업로드 통합 명령어 방식은 사용하지 않는다.
- 한 줄씩 복사해서 입력할 수 있게 제공한다.
- 모바일 기준 폴더는 `/root/apps/mobile_app`이다.
- 서버 기준 폴더는 `/root/apps/green_api`이다.

## 주의사항

- 기능 조건, 권한 조건, 저장 조건, 동기화 조건, 계산 조건은 임의로 바꾸지 않는다.
- UI 수정은 화면 배치, 여백, 크기, 색상, 버튼 위치, 표시 방식 중심으로 한다.
- 서버 방어 로직과 모바일 화면 숨김을 혼동하지 않는다.
- 민감정보는 저장하지 않는다.
