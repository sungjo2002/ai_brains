# workforce 지식

이 파일은 인력관리 프로그램의 고정 규칙, 기본 명령어, 주소 기준, 파일명 규칙, 판단 기준을 기록한다.

## 저장소 기준

- 저장소: `sungjo2002/ai_brains`
- 앞으로 대화에서 별도 설명 없이 “저장소”라고 하면 GitHub 저장소 `sungjo2002/ai_brains`를 의미한다.
- 작업내용은 가능한 한 저장소에 저장한다.
- 모든 사적인 대화가 아니라 실제 작업 기준/작업 기록/완료 내용/다음 작업만 저장한다.

## 파일명 규칙

- PC 수정본: `pc_번호_영문설명.zip`
- 모바일 수정본: `mobile_번호_영문설명.zip`
- 서버 수정본: `server_번호_영문설명.zip`
- PC/모바일/서버 수정본은 각각 따로 만든다.
- ZIP 안에는 불필요한 최상위 바깥 폴더를 만들지 않는다.
- `__pycache__`, `.pyc` 파일은 제거한다.

## 수정 전 규칙

실제 수정/생성 요청이 있으면 먼저 아래 형식으로 짧게 정리한다.

```text
수정 대기
- 수정 대상:
- 수정 이유:
- 수정 범위:
- 생성/수정 파일명:
- 확인 방법:
```

그 다음 실제 작업을 진행한다.

## 주의사항

- 기능 조건, 권한 조건, 저장 조건, 동기화 조건, 계산 조건은 임의로 바꾸지 않는다.
- UI 수정은 화면 배치, 여백, 크기, 색상, 버튼 위치, 표시 방식 중심으로 한다.
- 서버 방어 로직과 모바일 화면 숨김을 혼동하지 않는다.
- 민감정보는 저장하지 않는다.
