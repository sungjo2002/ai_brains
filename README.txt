3. 로그인창 진행바 애니메이션 제거.
4. 조각/잔상 깜박임 완료 처리.
5. 캐시 파일 정리 완료.
6. 화면 품질 작업은 이번 기준 여기까지 완료 처리.

### 2-2. 근태관리 화면 레이아웃 완료

2. 근태표 영역 안에서만 필요한 스크롤이 보이도록 화면 비율 정리.
3. 근태 저장/서버 동기화/권한/마감/계산 로직은 건드리지 않음.

### 2-3. 설정창 검색/보기 크기 정리 완료

1. 설정창 검색창과 보기/선택창 크기 통일.
2. 검색창 폭/높이와 보기창 폭/높이 균형 정리.

완료 기준 파일:
- pc_13_settings_search_view_size_fix.zip

### 2-4. 사원등록 계좌정보 추가 완료

1. 사원등록 화면 문서 자동추출 정보 영역에 계좌정보 입력 추가.
2. 계좌번호 앞에 은행 선택칸 추가.
3. 은행은 선택 방식 콤보박스 사용.
4. 은행 목록에 직접입력 항목 포함.
5. 신규 사원 등록 시 은행명 + 계좌번호 저장.
6. 기존 사원 사진/문서 수정 시 은행명 + 계좌번호 불러오기/저장.
7. 근로자 관리 수정 팝업에서도 은행명 + 계좌번호 수정 가능.
8. 저장 키는 bank_name / bank, bank_account / account_number 흐름 유지.

완료 기준 파일:
- pc_15_employee_bank_name_fields.zip

### 2-5. 급여명세표 계좌 확인 표시 완료

1. 급여명세표 미리보기 하단 버튼 영역에 지급 전 확인용 입금계좌 표시 추가.
2. 출력되는 급여명세표 문서 안쪽에는 계좌정보를 넣지 않음.
3. 표시 형식: 입금계좌: 은행명 계좌번호 예금주.
4. 은행명 또는 계좌번호가 없으면 미등록 안내 표시.
5. 다시 생성 버튼을 눌러도 계좌정보 갱신.

완료 기준 파일:
- pc_16_paystub_bank_info_bottom_check.zip

### 2-6. 급여명세표 이전/다음 직원 버튼 완료

1. 급여명세표 미리보기 화면 좌우에 이전/다음 직원 버튼 추가.
2. 현재 급여관리 화면에 표시된 직원 순서 기준으로 이동.
3. 첫 직원에서 이전을 누르면 마지막 직원으로 이동.
4. 마지막 직원에서 다음을 누르면 첫 직원으로 이동.
5. 직원 이동 시 급여명세표 내용, 창 제목, 하단 입금계좌 정보가 같이 갱신됨.
6. 이동한 직원은 급여관리 표에서도 선택 위치가 맞춰짐.
7. 버튼 모양은 세로형/가느다란 버튼으로 수정.
8. 버튼 크기 기준: 폭 26px, 높이 150px.

완료 기준 파일:
- pc_18_paystub_slim_vertical_nav_buttons.zip

### 2-7. 기본 근무/급여 설정 변경 완료

1. 기본 근무형태: 주간 -> 교대.
2. 기본 급여형태: 월급제 -> 시급제.
3. 설정 화면 기본값, 설정 기본값 복원, 신규 사원 등록 기본값, 근로자 급여 설정 fallback 기본값에 반영.
4. 기존에 저장된 개인별 설정값과 공장별 설정값은 그대로 유지.
5. 급여 계산식은 수정하지 않음.

완료 기준 파일:
- pc_19_default_shift_hourly_settings.zip

## 9. 서버 적용 기본 명령어

서버 수정본 적용 후 확인:

cd /root/apps/green_api
python3 -m compileall -q app
systemctl restart green_api
systemctl status green_api --no-pager -l
curl -s http://127.0.0.1:8000/api/health

주의:
- /root/apps/green_api 전체 삭제 금지.
- .env, .env.bak, .env.bak2, .venv 유지.
- app/ 교체와 필요한 파일만 적용.

## 10. 모바일 적용 기본 명령어

서버에는 node가 없으면 node --check app.js는 서버에서 생략.

cd /root/apps/mobile_app
rm -rf /var/www/mobile_live/*
cp -r ./* /var/www/mobile_live/
chown -R www-data:www-data /var/www/mobile_live
find /var/www/mobile_live -type d -exec chmod 755 {} \;
find /var/www/mobile_live -type f -exec chmod 644 {} \;
systemctl restart nginx

## 11. 작업 규칙

- 실제 코드 수정은 사용자가 "수정해", "적용해", "파일 만들어줘", "ZIP 만들어줘", "만들어줘"처럼 명확히 말했을 때만 진행.
- "하자", "할까", "다음은", "정리하자"는 우선 분석/계획으로 처리.
- 실제 수정 전에는 먼저 "수정 대기"를 출력하고 수정 대상/이유/범위/생성 파일명/확인 방법을 짧게 정리한 뒤 진행.
- 기능 조건, 권한 조건, 저장 조건, 동기화 조건, 계산 조건은 임의로 바꾸지 않음.
- 모바일 삭제 기능은 임의로 추가하지 않음.
- 권한 제한은 화면 숨김뿐 아니라 서버 API에서도 검사되어야 함.
- OCR/문서보정/사진추출은 현재 확인 항목이 아니라 보강 항목으로 관리.
