# AGENTS.md

이 저장소는 사용자의 작업내용 저장소로 사용한다.

## 최우선 사용 규칙

- 다음부터 인력관리 프로그램 및 관련 프로젝트의 작업내용은 가능한 한 이 GitHub 저장소 `sungjo2002/ai_brains`에 저장한다.
- 여기서 말하는 저장 대상은 모든 사적인 대화가 아니라, 실제 작업 기준/작업 기록/완료 내용/다음 작업 같은 작업내용이다.
- 새 대화에서 작업을 이어갈 때는 먼저 아래 파일을 확인한다.
  - `index.md`
  - `log.md`
  - `workforce/지식.txt`
  - `workforce/README.txt`
- 민감정보는 저장하지 않는다. 예: 서버 비밀번호, 토큰, 개인식별번호, 비밀키.

## 파일 역할

- `workforce/지식.txt`: 고정 규칙, 서버 구조, 파일명 규칙, 배포 명령어, 판단 기준.
- `workforce/README.txt`: 현재 작업 상태, 완료/진행/보류/오류, 다음 작업, 테스트 결과.
- `index.md`: 저장소 문서 목록과 역할.
- `log.md`: 변경 이력. 날짜별로 append 방식으로 기록한다.

## 수정 전 규칙

사용자가 실제 수정/생성을 요청하면 먼저 다음 형식으로 짧게 말한다.

```text
수정 대기
- 수정 대상:
- 수정 이유:
- 수정 범위:
- 생성/수정 파일명:
- 확인 방법:
```

그 다음 실제 작업을 진행한다.

## 작업 저장 기준

- 완료된 작업은 `workforce/README.txt`에 완료 처리한다.
- 새로 확정된 고정 규칙은 `workforce/지식.txt`에 추가한다.
- 저장소 구조나 문서 목록이 바뀌면 `index.md`를 갱신한다.
- 의미 있는 저장/수정/정리 작업은 `log.md`에 날짜별로 기록한다.

## 인력관리 프로그램 수정본 파일명 규칙

- PC 수정본: `pc_번호_영문설명.zip`
- 모바일 수정본: `mobile_번호_영문설명.zip`
- 서버 수정본: `server_번호_영문설명.zip`
- PC/모바일/서버 수정본은 각각 따로 만든다.
- ZIP 안에는 불필요한 최상위 바깥 폴더를 만들지 않는다.
- `__pycache__`, `.pyc` 파일은 제거한다.

## 주의

- 기능 조건, 권한 조건, 저장 조건, 동기화 조건, 계산 조건은 임의로 바꾸지 않는다.
- UI 수정은 화면 배치, 여백, 크기, 색상, 버튼 위치, 표시 방식 중심으로 한다.
- 서버 방어 로직과 모바일 화면 숨김을 혼동하지 않는다.
